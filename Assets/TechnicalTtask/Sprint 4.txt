Введение в Unity и создание первой игры.

1. Перемещение оказалось действительно не очень удобным. Давай сделаем дополнительную функцию для игрока, 
которую пока будем включать через поле бул в инспекторе. Пусть наш корабль будет иметь 2 типа управления:
1) Управление как есть сейчас
2) Управление при котором вперед и назад будут считываться относительно положения корабля, т.е. локальные. 
Если нос корабля повернут вниз, то при нажатии кнопки вперед - W или 8 корабль будет лететь на самом деле в низ. И наборот.

Дополнительные изучение C# и Unity API:
1. Продолжить предыдущие задание спринта 4. Каждый вектор сложить с предыдущим. Первый соотвественно не складывать ни с кем. 
После вычисления позиций, нужно удалить из списка все вектора превышающие по x > 5.
2. У нас есть игровой персонаж. Он находится в позиции Vector3 == x. 
Напишите код при котором перосонаж по нажатию на пробел прыгнет на Vector3 y. (Использовать векторную арифметику). 
Для ввода x и y использовать сериализацию приватных переменных. Результат его новой позиции вывести в консоль.
3. У нас есть игровой персонаж. Он находится в позиции Vector3 == x. 
Напишите код при котором перосонаж по нажатию на W сдвинется на Vector3 y с параметром ускорения, заданными в поле инспектора. 
(Использовать векторную арифметику). Для ввода x и y использовать сериализацию приватных переменных. 
Результат его новой позиции вывести в консоль.
4. Написать программу считающая косинус Vector3 == x. Для ввода x использовать сериализацию приватных переменных. 
Результат вывести в консоль.
5. Создай отдельную сцену для тренировки АПИ юнити. На нее добавь новый Test объект - примитив Cube. 
Для этого объекта создай скрипт, который по нажатию на пробел будет включать и выключать этот объект в зависимости от того, включен он или нет.
6. Для объекта Test создай компонент, который будет по нажатию на enter включать и выключать компонент из задачи 5. 
Т.е. после того как нажали на Enter объект №5 перестает работать, следовательно больше не реагирует на клавишу пробел. 
(Следует изучить способы доступа к компонентам объекта)
7. Напиши еще 3 дополнительных способа для доступа к объекту из задания 6.
8. Напиши компонент, который будет уничтожать твоей Test объект по нажатию на кнопку D.
